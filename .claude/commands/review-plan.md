name: review-plan
description: Review a plan for feature/system design as a senior developer
---

Review the plan as a senior developer.

Review it for things like: internal consistency, accuracy, unhandled edge cases, ambiguous requirements, inflexibility, poor architecture, inability to handle likely future requirements, overengineering, adherence to established patterns.

- This list is not exhaustive though. Think deeply about what a senior engineer would flag.
- Deep multi-stage thinking will be required to achieve this.
- Ensure key architectural patterns that already exist in the project have been followed, all that adequate justification has been given for breaking them.
- Ensure that project knowledge in the Claude.md file has been read and respected.
- Ask clarifying questions if needed, as a senior developer would.

Once you are ready:

- Present your review in a clear, structured format. 
- Categorise your findings and recommend improvements. 
- Ask clarifying questions if needed.
- All questions and confirmations should be presented in a numbered list so the user can reply directly to them.

Work towards an optimal plan with the user. This may require multiple iterations of review and revision.

If the identity of the plan document is not obvious, then this command may have been invoked by accident. Clarify with the user before proceeding.
